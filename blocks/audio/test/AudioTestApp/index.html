This app tests browser audio playback and microphone input. Audio starts only after a user click.

  • Play local MEMFS WAV: plays a generated test tone from the in-memory filesystem.
  • Play URL or default WAV: plays the URL in the input field, or /tone.wav if the field is empty.
  • Start microphone loopback: routes AudioSource -> AudioSink so you can hear your microphone and see live log output.

Remote WAV URLs are fetched through the local /proxy endpoint. Use headphones for microphone loopback unless you want acoustic feedback.
